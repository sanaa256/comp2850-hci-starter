## Story S1: Confirmation Feedback
**Situation**: When I submit a form (add/edit/delete task)
**Motivation**: I want immediate, explicit confirmation that the action has succeeded
**Outcome**: So I can trust the interface without refreshing to verify
**Underlying need**: Because uncertainty about save status causes anxiety, organisational slip-ups, and inefficient workarounds

**Evidence**: Participant A (notes line 66), Participant B (notes lines 122, 126 )
**Inclusion risk**: Cognitive, Screen reader, Low digital literacy
**Type**: Job story
**WCAG**: 4.1.3 Status Messages (AA)

## Story S2: Text Size Adjustment
**Situation**: When I'm reading over my task list
**Motivation**: I want to be able to adjust text size within the app
**Outcome**: So I can read it clearly without squinting
**Underlying need**: Because excessive squinting and coming too close to the screen cause eye strain and distress, and I could also misread a task

**Evidence**: Participant A (notes line ), Participant B (notes line )
**Inclusion risk**:
**Type**: Job story
**WCAG**: 1.4.4 Resize Text (AA)

## Story S3: Keyboard Interface Operability
**Situation**: When I'm navigating my tasks (reading/adding/editing/deleting)
**Motivation**: I want to be able to perform these operations using a keyboard interface
**Outcome**: So I can navigate the screen without a mouse or touchpad
**Underlying need**: Because it enables me to use my screen reader. Also, pressing keys doesn't cause as much strain as a mouse and requires less precise movements.

**Evidence**: Participant A (notes line 48), Participant B (notes line 135)
**Inclusion risk**: Cognitive, Motor, Screen reader
**Type**: Job story
**WCAG**: 2.1.1 Keyboard (A)
